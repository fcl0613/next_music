## 1. 项目概述

### 1.1 项目背景

将 B 站中收藏的音乐视频（MV、纯音乐、翻唱等）通过 BV 号一键收录到本地数据库，建立个人音乐库，并提供歌单管理与在线播放功能，解决 B 站音乐分散、不便集中收听的问题。

### 1.2 核心价值

- **一键收录**：输入 BV 号自动解析视频信息入库，无需手动下载
- **集中管理**：将分散在 B 站收藏夹的音乐聚合为个人音乐库
- **流畅播放**：内置播放器 + 播放列表，体验接近本地音乐播放器

### 1.3 目标用户

个人使用者（单用户系统）。架构上预留多用户扩展能力，但 v1 仅实现单用户。

## 2. 功能需求

### 2.1 功能模块总览

| 模块 | 核心功能 | 优先级 |
|------|---------|--------|
| 音乐收录 | BV 号解析、信息入库、批量收录 | P0 |
| 音乐库管理 | 列表浏览、搜索、编辑、删除 | P0 |
| 歌单管理 | 创建歌单、加入/移出、拖拽排序 | P0 |
| 播放器 | 播放控制、播放列表、循环模式 | P0 |
| 音频代理 | 解决 B 站防盗链与 CORS 限制 | P0 |
| 设置 | B 站 Cookie 配置（浏览器本地存储） | P1 |

### 2.2 音乐收录（P0）

**FR-2.2.1 单个收录**
- 用户输入一个 BV 号（支持带 `BV` 前缀或不带）
- 系统调用 B 站视频信息接口解析元信息：标题、封面、UP 主、时长、分 P 信息
- 将音乐信息写入数据库，状态标记为「已收录」
- **收录时不解析音频流**，音频流在播放时按需获取（见 2.6 音频代理）
- 若该 BV 号已收录，提示用户而非重复录入

**FR-2.2.2 批量收录**
- 用户可输入多个 BV 号（换行或逗号分隔）
- 系统逐个解析视频信息入库，返回成功/失败明细
- 失败原因：视频不存在、需要登录、被下架等
- 由于仅调用视频信息接口（不调播放地址接口），收录速度更快、被风控风险更低

**FR-2.2.3 分 P 处理**
- 若视频有多个分 P（如音乐合集），列出所有分 P 供用户选择
- 每个分 P 作为独立音乐条目入库，关联同一 BV 号

**FR-2.2.4 收录信息补充**
- 用户可手动编辑标题、艺人、封面（B 站标题通常不规范）
- 支持自定义标签（如「纯音乐」「翻唱」「OST」）

### 2.3 音乐库管理（P0）

**FR-2.3.1 列表浏览**
- 表格 / 卡片两种视图切换
- 显示：封面缩略图、标题、UP 主/艺人、时长、收录时间、所属歌单
- 支持按标题、收录时间、时长排序

**FR-2.3.2 搜索与筛选**
- 关键词搜索（标题、UP 主、标签）
- 按标签筛选
- 按歌单筛选

**FR-2.3.3 编辑与删除**
- 编辑音乐元信息（标题、艺人、标签、封面）
- 删除音乐（同时从所有歌单移除）
- 删除前二次确认

### 2.4 歌单管理（P0）

**FR-2.4.1 歌单 CRUD**
- 创建歌单（名称、描述、封面）
- 编辑歌单信息
- 删除歌单（仅删除歌单关系，不删除音乐本身）

**FR-2.4.2 歌单内容管理**
- 将音乐加入歌单（支持多选批量加入）
- 从歌单移除音乐
- 歌单内音乐拖拽排序
- 歌单内搜索

**FR-2.4.3 默认歌单**
- 系统初始化时创建「我喜欢的音乐」默认歌单
- 收录新音乐时可快速加入默认歌单

### 2.5 播放器（P0）

**FR-2.5.1 播放控制**
- 播放 / 暂停
- 上一首 / 下一首
- 进度条拖动跳转
- 音量调节
- 播放速率（0.75x / 1x / 1.25x / 1.5x）

**FR-2.5.2 播放模式**
- 顺序播放
- 单曲循环
- 随机播放
- 列表循环（默认）

**FR-2.5.3 播放列表**
- 「播放全部」：将当前歌单/筛选结果加入播放队列
- 「添加到播放列表」：单曲加入队列
- 播放列表内拖拽排序、删除单项、清空列表
- 当前播放高亮显示

**FR-2.5.4 播放持久化**
- 刷新页面后恢复播放队列与当前播放位置（localStorage）

### 2.6 音频代理（P0）

B 站音频流地址有防盗链（Referer 校验）且存在 CORS 限制，浏览器无法直接播放。需要后端代理转发音频流。**音频流在播放时按需获取，不提前收录。**

- 提供 `/api/audio/[id]` 代理接口
- 后端根据 music 记录的 bvid + cid 调用 B 站播放地址接口获取音频流 URL
- 从请求头 `X-Bili-Cookie` 读取用户配置的 Cookie，用于访问需要登录的视频
- 携带 `Referer: https://www.bilibili.com` 请求 B 站音频
- 流式转发给前端，支持 Range 请求（进度跳转）
- 获取到的音频 URL + 过期时间缓存到数据库，下次播放时若未过期则直接使用缓存

### 2.7 设置（P1）

**FR-2.7.1 B 站 Cookie 配置**
- 配置 SESSDATA 等 Cookie，用于解析需要登录才能访问的视频
- Cookie 存储在浏览器 **localStorage** 中，不存入数据库
- 前端在每次调用后端 API 时通过请求头 `X-Bili-Cookie` 传递给服务端
- 后端从请求头中读取 Cookie，用于调用 B 站 API

**FR-2.7.2 播放偏好**
- 默认音质选择、默认播放模式，同样存储在浏览器 localStorage

### 3.2 请求流转

**收录流程：**
```
用户输入BV号
  → Next.js API Route 接收
    → Service 层调用 B 站视频信息接口获取视频元信息
      → 写入 PostgreSQL（music 表，仅存视频信息，不含音频流）
        → 返回收录结果
```

**播放流程：**
```
用户点击播放
  → 播放器请求 /api/audio/[id]
    → 后端查询数据库是否有缓存的 audio_url 且未过期
      → 有缓存 → 直接使用
      → 无缓存/已过期 → 调 B 站播放地址接口获取新 URL，更新缓存
        → 携带 Referer 头请求 B 站音频流
          → 流式转发给浏览器 <audio> 元素
```

## 6. B 站解析方案

### 6.1 收录解析流程（仅视频信息）

```
输入 BV 号
  │
  ▼
调用视频信息接口
   GET https://api.bilibili.com/x/web-interface/view?bvid={bvid}
   → 获取 aid, cid, title, pic, owner.name, duration, pages[]
  │
  ├─ 单 P 视频 → 直接使用主 cid
  └─ 多 P 视频 → 返回分 P 列表供用户选择
  │
  ▼
写入数据库（music 表）
   仅存储视频元信息，不调用播放地址接口，不获取音频流
```

**收录时仅需一次 API 调用**，相比同时解析音频流：
- 收录速度更快，批量收录效率提升一倍
- 减少 B 站 API 请求量，降低被风控风险
- 避免收录后不立即播放导致音频 URL 过期浪费

### 6.2 播放解析流程（音频流按需获取）

```
用户点击播放 /api/audio/[id]
  │
  ▼
① 查询数据库是否有缓存的 audio_url
   ├─ 有缓存且未过期 (audio_expire > now) → 直接使用缓存的 URL
   └─ 无缓存或已过期 → 进入步骤 ②
  │
  ▼
② 调用播放地址接口（DASH 格式）
   GET https://api.bilibili.com/x/player/playurl?bvid={bvid}&cid={cid}&fnval=16&fnver=0
   → 获取 data.dash.audio[] 音频流列表
  │
  ▼
③ 选择音频流
   从 audio 数组中选择指定音质的 baseUrl
   音质优先级（由高到低）：
     30250 → Hi-Res / Dolby
     30280 → 192K 高品质
     30232 → 132K 标准
     30216 → 64K 低品质
  │
  ▼
④ 更新数据库缓存
   写入 audio_url + audio_quality + 过期时间（当前时间 + 2 小时）
  │
  ▼
⑤ 代理转发音频流（见 6.4）
```

### 6.3 关键注意事项

| 问题 | 方案 |
|------|------|
| **音频地址时效性** | B 站音频 URL 有效期约 2 小时。播放时获取并缓存到 `audio_expire`，下次播放若未过期则复用缓存，过期则重新获取 |
| **防盗链 (Referer)** | 浏览器直接请求会被 403。后端代理时携带 `Referer: https://www.bilibili.com` + `User-Agent` |
| **CORS 限制** | 浏览器无法跨域请求 B 站音频。必须走后端 `/api/audio/[id]` 代理 |
| **WBI 签名** | 部分 API 需要 WBI 签名。视频信息接口目前可用，播放地址接口建议带上签名以防风控 |
| **需要登录的视频** | 配置 SESSDATA Cookie（存储在浏览器 localStorage）。前端通过请求头 `X-Bili-Cookie` 传递给后端，后端在调用 B 站 API 时携带 `Cookie` 头 |
| **Range 请求** | 代理需支持 `Range` 请求头透传，实现进度拖动跳转 |
| **视频下架** | 播放时获取音频流失败，标记 status=`invalid`，列表中灰显 |
| **首次播放延迟** | 首次播放需调用播放地址接口获取音频流，有约 200-500ms 额外延迟。后续播放复用缓存则无此延迟 |

### 6.4 音频代理实现要点

```typescript
// app/api/audio/[id]/route.ts（伪代码）
export async function GET(req: Request, { params }: { params: { id: string } }) {
  const music = await getMusicById(params.id);

  // 从请求头读取用户配置的 Cookie
  const biliCookie = req.headers.get('x-bili-cookie') || '';

  // 1. 检查数据库缓存是否有效
  if (!music.audioUrl || isExpired(music.audioExpire)) {
    // 无缓存或已过期 → 调 B 站播放地址接口获取新音频流
    const newUrl = await fetchAudioStreamUrl(music.bvid, music.cid, preferredQuality, biliCookie);
    // 更新数据库缓存
    await updateMusicAudioCache(music.id, {
      audioUrl: newUrl.url,
      audioQuality: newUrl.quality,
      audioExpire: new Date(Date.now() + 2 * 60 * 60 * 1000), // 2 小时后过期
    });
    music.audioUrl = newUrl.url;
  }

  // 2. 代理请求 B 站音频流
  const range = req.headers.get('range');
  const headers: Record<string, string> = {
    Referer: 'https://www.bilibili.com',
    'User-Agent': 'Mozilla/5.0 ...',
  };
  if (range) headers['Range'] = range;

  const audioRes = await fetch(music.audioUrl, { headers });

  // 3. 流式转发，透传 Range 响应
  return new Response(audioRes.body, {
    status: audioRes.status,
    headers: {
      'Content-Type': 'audio/mp4',
      'Accept-Ranges': 'bytes',
      'Content-Length': audioRes.headers.get('content-length') || '',
      'Content-Range': audioRes.headers.get('content-range') || '',
    },
  });
```

## 11. 风险与注意事项

### 11.1 技术风险

| 风险 | 影响 | 应对 |
|------|------|------|
| B 站 API 变更或风控 | 收录功能不可用 | API 调用层抽象封装，便于切换；监控失败率 |
| 音频地址频繁过期 | 播放中断 | 播放时检查缓存过期时间，自动重新获取；缓存复用减少请求 |
| 首次播放有延迟 | 用户体验 | 首次播放需调播放地址接口，约 200-500ms；可预加载下一首的音频流 |
| B 站反爬升级 | 批量收录被限流 | 收录仅调一次 API（视频信息），请求量已减半；控制频率 + 重试 |
| WBI 签名要求 | API 调用失败 | 实现签名工具，预留开关 |
| 视频版权下架 | 已收录音乐无法播放 | 标记 invalid 状态，保留记录，定期检查 |

### 11.2 法律与合规

- 本项目仅供**个人学习使用**，不对外提供服务
- 收录的音频流地址**不下载存储**到本地，仅代理转发
- B 站音频版权归原 UP 主所有，不得用于商业用途
- 遵守 B 站 robots.txt 与 API 使用规范，控制请求频率

### 11.3 性能注意

- 音频代理是性能瓶颈，高并发时需考虑连接池
- 音乐列表分页查询，避免全量加载
- 封面图片走 B 站 CDN，可直接加载（需处理防盗链）
- 播放列表数据量大时，前端虚拟滚动

```prisma
model Music {
  id            BigInt   @id @default(autoincrement())
  bvid          String   @db.VarChar(20)
  aid           BigInt?
  cid           BigInt
  pageIndex     Int      @default(1) @map("page_index")

  audioQuality  Int?     @map("audio_quality")          // 播放时缓存
  audioExpire   DateTime? @map("audio_expire")          // 播放时缓存，2 小时过期
  status        String   @default("active") @db.VarChar(20)

  updatedAt     DateTime @updatedAt @map("updated_at")

  collectionItems CollectionItem[]
  tags            MusicTag[]

  @@unique([bvid, pageIndex])
  @@index([title])
  @@index([artist])
  @@index([status])
  @@map("music")
}

model Collection {
  id          BigInt   @id @default(autoincrement())
  name        String   @db.VarChar(100)
  description String?  @db.Text
  coverUrl    String?  @db.VarChar(500) @map("cover_url")
  isDefault   Boolean  @default(false) @map("is_default")
  sortOrder   Int      @default(0) @map("sort_order")

}
```
